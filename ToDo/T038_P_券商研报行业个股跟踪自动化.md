# T038 券商研报行业个股跟踪自动化

| 属性 | 值 |
|------|-----|
| 状态 | P (未开始) |
| 类型 | automation |
| 分类 | 研报筛选 + 跟踪 + 建议 |
| 创建日期 | 2026-05-21 |

---

## 一、策略概要

券商研报大量产出行业/个股推荐，但质量和时效性参差不齐。目标是建立一条 **阅读 → 理解 → 筛选 → 跟踪 → 买卖建议** 的自动化管道。

核心区别于传统量化：这里做的是 **研报信号 → 量化验证**，而非纯价格信号。

---

## 二、管道设计

### 阶段 1：阅读（数据采集）
- akshare `stock_research_report` 获取最新研报
- 券商覆盖：中金、中信、华泰、天风、海通等前 10 大
- 每日/每周增量

### 阶段 2：理解（NLP/关键词提取）
- 目标行业：识别"明确方向"（推荐/看好/增持）
- 个股：识别目标价变化（上调/下调）
- 关键催化：识别"拐点""反转""底部""超预期"
- 摘要生成：每份研报 2~3 句话的核心观点

### 阶段 3：筛选（规则过滤）
- 剔除：中性/观望评级、无明确方向
- 筛选：上调盈利预测 + 目标价 > 当前价 20%+
- 筛选：首次覆盖（市场关注度低 → 信息不对称价值高）
- 筛选：行业拐点信号（多家券商同时看好同一行业）

### 阶段 4：跟踪（量化回测）
- 对筛选出的股票做历史信号验证：
  - 类似推荐历史（券商推荐该股后 X 日的胜率）
  - 行业拐点信号历史（过去 5 年同类信号的有效比例）
- 生成跟踪卡片：代码/简称/信号类型/时效/置信度

### 阶段 5：买入卖出建议
- 买入：筛选通过的标的 + 价格在 MA60 上方（趋势确认）
- 卖出：券商下调评级 / 目标价下调 20%+ / 跌破 MA60 且成交量萎缩

---

## 三、技术挑战

| 挑战 | 方案 |
|------|------|
| 研报质量不一 | 只取前 10 券商 + 只取"强力推荐"以上 |
| NLP 准确率 | 先用关键词规则（rapidfuzz），后续可接 LLM |
| 股票池不是 ETF | 需要个股数据源（akshare 有 A 股日线） |
| 推荐时效性 | 发布日 vs 我们拿到日的时间差 |
| 信息不对称已消失 | "首次覆盖"可能在发布前已被市场消化 |

---

## 四、MVP 范围

- 先只做"行业方向信号"（不选个股），降低复杂度
- 用 `akshare.stock_research_report` 获取标题+摘要
- 关键词匹配提取行业 + 方向
- 人工关注 + 量化验证（不做自动交易）

---

## 五、与现有策略关系

- 完全独立——输入是研报文本，输出是行业/个股信号
- 可能的交集：研报信号确认的行业 → 加入 ROC 动量的候选 ETF 池？
