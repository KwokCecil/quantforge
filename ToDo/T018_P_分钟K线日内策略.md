### Step 1：实现 IntradayConfig

```python
@dataclass
class IntradayConfig(StrategyConfig):
    etf_code: str = "510300"
    decision_time: str = "14:30"       # 信号判断时间
    execute_time: str = "14:55"        # 执行时间
    half_position: bool = True         # 半仓模式
    lookback_days: int = 5             # 涨跌判断回看期
    trend_threshold: float = 0.003     # 涨跌幅阈值（>0.3%才行动）
```

### Step 2：实现 IntradayStrategy

```python
class IntradayStrategy(Strategy):
    """
    日内半仓策略。
    14:30 判断：(今日涨幅 × 近期趋势) → enter/exit/hold
    14:55 执行：产出 TIMING Decision
    """

    def produce_decisions(self, data: DataResponse) -> list[Decision]:
        minute_data = data.bar_data[self._config.etf_code]
        # 只用到 14:30 之前的数据做判断
        recent = minute_data[minute_data['date'].dt.time <= time(14, 30)]

        today_return = recent['close'].iloc[-1] / recent['close'].iloc[0] - 1
        trend = self._calc_trend(recent)

        if today_return > self._config.trend_threshold and trend > 0:
            direction = 'enter'
        elif today_return < -self._config.trend_threshold and trend < 0:
            direction = 'exit'
        else:
            direction = 'hold'

        return [Decision(
            decision_type=DecisionType.TIMING,
            direction=direction,
            target_code=self._config.etf_code,
            weight=0.5 if self._config.half_position else 1.0,
        )]
```

## 六、备注

- **为什么优先级低**：日内策略的资金利用率只提升 ~2%（半仓过夜 vs 尾盘买入次日开盘），但复杂度（分钟数据处理、时间对齐、尾盘流动性）大幅增加
- **可替代方案**：如果分钟 K 线不支持，用日 K 线的收盘价做"伪日内"判断——逻辑相同但用日频数据。不过这样就不是真正的"日内"了
- **实际下单的延迟**：14:55 下单 → 盘后成交（集合竞价），无法保证成交在当日收盘价。实际执行中可能产生额外滑点
