## 三、实施计划

### Step 1：接入真实 PE 数据，替换 MA 比率估值

**数据源选择**：akshare `index_value_name_fund_()` → 沪深300 PE，日频，免费

**估值信号改造**：

```
旧: ERP_approx = -(close/MA200 - 1) × 100 + 国债收益率
新: ERP_real   = 1/PE_沪深300 × 100 - 10Y国债收益率
               = 盈利收益率 - 无风险利率
```

**实施要点**：
1. 在 `BondYieldConfig` 中增加 `use_real_pe: bool = True` 开关，默认启用
2. `BondYieldStrategy.produce_decisions()` 中双路径：开关打开走 PE 路线，关闭走 MA 路线（向后兼容）
3. 加载 PE 数据失败时，降级为 MA 模式并 `logger.warning`
4. PE 数据通过 akshare 一次性加载历史序列（回测场景），或通过新增 DataFeed 加载（实盘场景）

### Step 2：接入真实国债收益率历史数据

**数据源选择**：akshare `bond_china_yield()` → 10Y 国债收益率，历史日频，一行代码

**实施要点**：
1. 回测时用 akshare 直接拉历史序列（一次性加载）
2. WebScraperFeed 保持"获取当日最新值"的定位，仅用于实盘监控
3. `DataRequest` 中可指定 `data_type="bond_yield"` 区分数源

### Step 3：增加 ERP 绝对水平过滤

分位数只看相对高低，不看绝对吸引力。增加硬阈值：

```python
ERP_ABS_MIN = 2.0   # ERP < 2% → 股票相对债券无明显优势
ERP_ABS_MAX = 6.0   # ERP > 6% → 极度低估，无视分位数直接买入
```

**逻辑**：
- 即使分位在买入区，但 ERP < 2%，降权至 0.5
- ERP > 6% 且分位在 50% 以上，视为强买入

### Step 4：验证新信号的统计有效性

- 用 T001 FactorLab 对真实 ERP 信号做 IC/ICIR 分析（替换原来的 MA 比率因子）
- 与旧信号（MA 比率）做 IC 对比，确认真实 PE 信号的增量预测力
- 用 T002 Validator 跑参数扫描，确认默认参数在新信号下是否仍合理
