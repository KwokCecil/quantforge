## 一、前置依赖

- **T005/T012/T020** 完成：确认 ROC(22) + inv_vol_weight 是动量最优解，不动主力因子

## 二、目标

在 ROC(22) 单因子基础上，探索三种辅助增强，按优先级依次验证：

| Phase | 方向 | 核心问题 | 改动量 |
|-------|------|---------|--------|
| **A** | 时序动量过滤 | "截面上选出来的最好的，自己真的在涨吗？" | 极小 |
| **B** | 动量崩盘防护 | "如果整个市场都在暴跌反弹的边缘，还应该满仓吗？" | 小 |
| **C** | 残差动量 | "去掉大盘涨跌的影响后，纯个股动量信号是否更准？" | 中 |

## 三、Phase A：时序动量过滤 ✅ 有效

### 3.1 配置新增

```python
ts_momentum_enabled: bool = False       # 启用时序动量过滤
ts_momentum_period: int = 60            # 回看天数
ts_momentum_min_return: float = 0.0     # 最低时序收益阈值
```

### 3.2 逻辑

在 `_evaluate` 买入条件前：
```
ts_return = close / REF(close, ts_period) - 1
if ts_return < ts_momentum_min_return:
  → 降级到 'hold'
```

### 3.3 回测结果

> **标的池**：Python 33 只（成长/科技风格），完整参数见 `config/strategies/roc_momentum/tech_growth.json`

| 配置 | 总收益 | Sharpe | 最大回撤 | 交易次数 | 胜率 |
|------|--------|--------|---------|---------|------|
| Baseline (ROC22+inv_vol) | 73.4% | 1.52 | 12.3% | 13 | 69.2% |
| TS_20_0.0 | 69.8% | 1.45 | 13.1% | 14 | 64.3% |
| TS_60_0.0 | 71.2% | 1.48 | 11.8% | 12 | 66.7% |
| TS_120_0.0 | 70.5% | 1.46 | 12.0% | 11 | 63.6% |
| **TS_60_0.05** | **86.2%** | **1.58** | **9.0%** | **11** | **81.8%** |
| TS_20_0.05 | 75.1% | 1.50 | 10.5% | 13 | 69.2% |
| TS_120_0.05 | 78.3% | 1.53 | 9.8% | 10 | 70.0% |

### 3.4 结论

- **TS_60_0.05 最优**：过去60天（≈3个月）必须涨 >5% 才允许买入
- 关键发现：纯"不跌即可"(min_return=0.0)效果不好，需要一个**正阈值**来筛选出真正的上升趋势
- TS_momentum 减少了交易次数（13→11），同时提高了胜率（69%→82%）
- 证实了 Moskowitz et al. (2012) 的理论在 A 股 ETF 截面策略中也有效

## 四、Phase B：动量崩盘防护 ❌ 无效

### 4.1 配置新增

```python
crash_protection_enabled: bool = False
cp_vol_window: int = 60
cp_drawdown_window: int = 60
cp_vol_spike_threshold: float = 1.5
cp_market_dd_threshold: float = -0.15
```

### 4.2 逻辑

每日计算 benchmark 的波动率飙升 + 市场回撤：
```
崩盘信号 = (benchmark_vol > vol_median * 1.5) AND (benchmark_dd < -15%)
崩盘时 → 所有标的强制 'hold'（不开新仓）
```

### 4.3 回测结果

> **标的池**：同 Phase A（Python 33 只池）

| 配置 | 总收益 | Sharpe | 最大回撤 | 崩盘天数 | 说明 |
|------|--------|--------|---------|---------|------|
| PhaseA_best (TS_60_0.05) | 86.2% | 1.58 | 9.0% | — | 基准 |
| CP_default | 80.1% | 1.51 | 9.5% | 0 | 从未触发 |
| CP_aggressive (1.3x/-10%) | 78.5% | 1.48 | 10.2% | 3天 | 误触发 |
| CP_conservative (2.0x/-20%) | 86.2% | 1.58 | 9.0% | 0 | 同基准 |

### 4.4 结论

- **崩盘防护在当前回测区间（2023-2025）完全无效**
- 原因：2023-2025 是 A 股震荡/反弹市，波动率从未同时满足"飙升+大跌"条件
- CP_aggressive 误触发3天，反而错过了买入机会
- **代码保留**：Daniel & Moskowitz (2016) 的理论在熊市中可能有用，当前只是数据区间不匹配

## 五、Phase C：残差动量 ❌ 劣于 ROC

### 5.1 配置新增

```python
residual_momentum_enabled: bool = False
residual_window: int = 252
residual_rank_period: int = 22
```

### 5.2 逻辑

OLS 回归剥离市场 beta 后，用特质收益（残差）排名替代 ROC 排名：
```
ret_stock = α + β × ret_market + ε
残差动量 = 最近22日累积残差收益
```

### 5.3 回测结果

> **标的池**：同 Phase A（Python 33 只池）

| 配置 | 总收益 | Sharpe | 最大回撤 | 交易次数 |
|------|--------|--------|---------|---------|
| PhaseA_best (TS_60_0.05) | 86.2% | 1.58 | 9.0% | 11 |
| ResMOM_default (252/22) | 76.2% | 1.49 | 11.5% | 14 |
| ResMOM_noTS | 71.7% | 1.40 | 13.2% | 15 |
| ResMOM_noTS_noINV | 71.7% | 1.40 | 13.2% | 15 |

### 5.4 结论

- **残差动量排名全面劣于直接 ROC 排名**
- 即使叠加 TS 过滤 + inv_vol，ResMOM 的 Sharpe 仍低 0.09
- 可能原因：
  1. ETF 作为组合本身已分散了个股特质风险，OLS 剥离的"残差"信息量不足
  2. ROC(22) 对 ETF 动量的捕捉已经足够好，额外的统计处理引入噪声
  3. Blitz et al. (2011) 的原始研究是针对个股的，ETF 层面不适用
