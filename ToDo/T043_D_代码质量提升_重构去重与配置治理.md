## 三、实施计划

### Step 1：止损逻辑提取（2h）

`RankingResolver`、`MacroOverlayResolver`、`TimingResolver` 中三处 `_check_stop_loss` 完全一致（~33行），提取为 `core/stop_loss.py` 的 `StopLossChecker` 类。

- 构造函数接收 `high_watermark_stop_edge`、`cut_loss_edge`
- 提供 `check(positions, data) -> list[TargetPosition]`
- 三个 Resolver 改为组合 `StopLossChecker` 实例

### Step 2：回测循环统一（4h）

`main_backtest.py` 的 `_run_rotation_backtest()` 与 `backtest_core.py` 的 `run_backtest()` 存在 ~80 行重复的回测主循环。将轮动调度逻辑抽象为 `RotationScheduler`，注入到 `run_backtest()` 中。

- 新建 `core/rotation_scheduler.py`：`RotationScheduler` 类，每日根据 `date` 返回 `(strategy, resolver, codes)`
- `run_backtest()` 新增可选参数 `rotation_scheduler`
- `_run_rotation_backtest()` 删除主循环，改为创建调度器 + 调用 `run_backtest()`

### Step 3：Monitor 数据管线共享（3h）

`monitors/roc_momentum_monitor.py` 与 `main_backtest.py` 在数据源初始化、缓存刷新、策略创建等环节重复。提取为 `core/pipeline.py`：

- `create_data_feed(cache_dir)` → `CachedDataFeed`
- `prepare_data(data_feed, codes, data_type, start, end)` → `DataResponse`
- 两者共用

### Step 4：ROCConfig 参数校验（3h）

为 `ROCConfig` 添加 `validate()` 方法，在 `__post_init__` 中自动调用，检查参数依赖一致性：

- `rsi_enhance_enabled=True` 但 RSI 相关参数未配置 → 抛 `ValueError`
- `style_rotation_enabled=True` 但 `sr_aggressive_preset` 为空 → 抛 `ValueError`
- 其他参数间依赖关系

### Step 5：僵尸参数清理（1h）

从 `ROCConfig` 和 JSON 预设中移除已被证伪无效果的参数：`sell_roc_edge`（★冗余）、`BUY_AVERAGE`（⚪无效果）、`REBALANCE`（⚪无效果）等。

### Step 6：Research 目录整理（1h）

`research/` 下 35+ 文件平铺，按职责分子目录：
- `research/tools/` — 可复用工具（`param_optimizer.py`、`validator.py` 等）
- `research/experiments/` — 一次性实验脚本
- `research/diagnostics/` — 诊断工具
