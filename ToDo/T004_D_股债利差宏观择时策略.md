## 二、目标

实现 `BondYieldStrategy`（股债利差宏观择时策略），产出 `TIMING` 类型 Decision。这将是框架中**第一个 TIMING 类型策略**，验证 `TimingResolver` 和多源数据接入（K线 + 爬虫宏观指标）的有效性。

## 三、实施计划

### Step 1：策略逻辑设计

**核心指标**：ERP（Equity Risk Premium，股权风险溢价）= 1/PE − 10年国债收益率

**简化版（L3 阶段）**：用标的 ETF 的 PE 近似或直接用指数 PE。如果无法直接获取 PE，用以下替代方案：

- **替代方案 A（推荐）**：用宽基指数 PE 数据（如沪深300 PE 来自公开数据源）
- **替代方案 B**：股债利差 = 沪深300 股息率 − 10年国债收益率
- **替代方案 C**：直接用沪深300 指数涨幅代替 ERP（粗糙但简单）

本任务采用**方案 A 或 B**，优先用可获取的公开 PE/股息率数据。

**择时规则**：

| 条件                | 决策        | 目标仓位                 |
| ----------------- | --------- | -------------------- |
| ERP > 历史 70 分位数   | 极度低估 → 买入 | target\_weight = 1.0 |
| ERP 在 30-70 分位数之间 | 正常区间 → 观望 | target\_weight 不变    |
| ERP < 历史 30 分位数   | 高估 → 卖出   | target\_weight = 0.0 |

### Step 3：实现 BondYieldStrategy

```python
# strategies/bond_yield.py

class BondYieldStrategy(Strategy):
    def produce_decisions(self, data, positions) -> list[Decision]:
        # 1. 从 data.macro_data 获取国债收益率
        # 2. 从 data.bar_data 获取股票 ETF K线（用于计算隐含 PE 或直接用）
        # 3. 计算 ERP = 1/PE - 国债收益率
        # 4. 计算 ERP 在历史中的分位数
        # 5. 根据分位数产出 TIMING Decision
        ...

    def get_required_data(self) -> list[DataRequest]:
        return [
            DataRequest(codes=[self.config.stock_code],
                       data_type='daily_k', ...),
            DataRequest(codes=[],
                       data_type='scraper_chinabond', ...),
        ]
```

### Step 4：数据组合方案

由于策略需要同时从 AutoStockFeed（K线）和 WebScraperFeed（国债收益率）获取数据，需要一种方式组合两个数据源：

**L3 简易方案**：在 `main_monitor.py` 或策略内部手动调用两个 Feed，合并为 DataResponse。

```python

stock_response = auto_feed.get_data(kline_request)

    bar_data=stock_response.bar_data,

```

## 七、2026-05 升级：真实 PE/国债数据 + 动量融合验证

### 背景

旧版 T004 使用**价格/均线比率模拟 PE**、**固定均值模拟国债收益率**。本次用 akshare 接入真实历史数据：
- HS300 PE（静态 + TTM）：`ak.stock_index_pe_lg("沪深300")`，2005年至今
- 10Y 国债收益率：`ak.bond_zh_us_rate()`，2002年至今

信号与旧工程 `临时文件/guzhai_licha.py` 一致：
- **滚动双倍股债利差** = 1/TTM_PE − 2×国债收益率
- **静态单倍股债利差** = 1/static_PE − 国债收益率
- 分位 = `reverse_ratio`（≥当前值的比例。低=便宜，高=贵）
- 冲锋 ≤15%，撤退 ≥92%

### 纯择时 510300 回测（2018-01 ~ 2026-05）

|------|:--:|:--:|
| 总收益率 | **+78.39%** | +16.25% |
| Sharpe | **0.55** | 0.19 |
| 最大回撤 | **-31.35%** | -45.10% |
| 交易次数 | 2买1卖（1持仓中） | — |

关键信号时段：
- 2019-01~2020-12：满仓吃满+69%

- 撤退仅在2021-01~03短暂触发（牛市末泡沫区）

### 动量仓位调节实验（证伪）

| 方案 | 总收益 | Sharpe | 回撤 | vs基线 |
|:--|:--:|:--:|:--:|:--:|
| 纯动量（基线） | +270.78% | 1.32 | 13.05% | — |
| 线性分段乘数 | +230.38% | 1.24 | 13.05% | -40pp |
| 仅撤退清仓 | +251.97% | 1.26 | 13.05% | -19pp |

**根因**：
1. 线性分段：2019-2020牛市期间 ratio 在 30~50%（中性区），乘数被压到 0.6~0.8，赚钱时踩刹车拖累收益
2. 仅撤退清仓：撤退信号（2021-01~03）与 RSI 过滤高度重叠，当时 RSI 已经阻止了所有新买，multiplier=0 对空仓无影响。与基线结果无实质差异

### 结论

- 股债利差 **纯择时 510300 有效**（+78% vs +16%），但信号极低频（8年仅2~3次交易）
- **不适合做动量的仓位调节器**：两个信号的时间维度不匹配（股债利差是年级别慢变量，动量是日级别）
- 撤退与 RSI 过滤功能重叠，无增量价值

## 八、低利率环境下信号改进探索（2026-05-22）

### 问题背景

国债利率从 2014 年的 4.5% 持续下行至当前 1.7%。利差公式（E/P − 国债）中，国债项逐年缩小，导致利差分布整体右移。扩展窗口分位法在低利率时代面临两个风险：

1. **敏感度不对称**：国债下行让利差偏宽，冲锋信号的触发频率可能越来越高（假阳性风险）
2. **校准滞后**：扩展窗口需要 5~10 年才能重新校准低利率环境

### 评估框架

用统一评估脚本 `research/_eval_guzhai_licha_signals.py` 对比 4 种信号定义：

| 维度 | 评估指标 | 标准 |
|------|---------|------|
| 1. 收益 | 159915择时总收益率 | ≥ 基线，至少不显著劣化 |
| 2. 风险 | Sharpe、最大回撤 | ≤ 基线 DD |
| 3. 语义正确 | 冲锋日 PE 中位数 | PE < 12（冲锋应反映"股便宜"） |
| 4. 2024-09 | 是否持仓经历大行情 | 最好已持仓 |
| 5. 当前建议 | 最新信号 | 合理（当前PE=13.6不应冲锋） |

### 四种信号定义

| 方案 | 方法 | 预期优势 |
|------|------|---------|
| 基线 | 利差全历史扩展窗口 reverse_ratio ≤ 15% | 当前方案 |
| 方案1 | 利差滚动5年窗口 reverse_ratio | 自我校准，适配当前利率 |
| 方案2 | E/P(1/PE)独立 reverse_ratio + 国债辅助 | 不受利率下行污染 |
| 方案3 | 利差 (x − 5y均值) / 5y标准差 Z-score | Z-Score天然自适应 |

### 回测对比（159915 择时，2018-01 ~ 2026-05）

| 指标 | 基线 | 方案1↗ | 方案2↗ | 方案3↗ | 买入持有 |
|------|:--:|:--:|:--:|:--:|:--:|
| 总收益率 | **+166.0%** | +150.1% | +151.2% | +139.8% | +131.0% |
| Sharpe | **0.57** | 0.55 | 0.56 | 0.54 | 0.49 |
| 最大回撤 | -46.8% | -47.0% | **-45.8%** | -45.7% | -56.6% |
| 冲锋天数 | 995 | 886 | 452 | 344 | — |
| 撤退天数 | 45 | 31 | 129 | 187 | — |

**冲锋日语义：**

| | 基线 | 方案1 | 方案2 | 方案3 |
|------|:--:|:--:|:--:|:--:|
| PE 中位数 | 11.2 | 11.1 | **10.7** | **10.7** |
| 国债中位数 | 2.65% | 2.65% | 2.83% | 2.66% |

**2024-09：** 四种方案均在 2024-09 期间无冲锋信号，均为"未持仓"状态。原因是：此期间 PE~13+，不在任一方案的"便宜"区间。

**当前（2026-05-21）：** 四种方案均为"中性"。

### 结论

1. **当前回测区间基线最优**（+166%），但不意外——扩展窗口基线包含该区间历史，改进方案为未见的低利率未来准备
2. **方案2（E/P独立分位）语义最优**：冲锋日 PE 中位数 10.7（最低），且不受国债下行直接污染；回撤也最低（-45.8%）
3. **方案3（Z-Score）最保守**：仅 344 天冲锋（17%），但收益仍优于持有
4. **暂不替换**：基线在 2018-2026 表现最佳，替换无直接收益。待国债降到 1.2% 以下时，方案2 作为备选
5. **关于 2024-09**：这不是信号能不能解决的问题——PE=13 在任何框架下都不算"极便宜"，只要"便宜才买"的前提不变，2024-09 就不会触发买入

## 九、动态分层仓位回测（2026-05-22）

### 目标

将基线"全仓进出"改为动态仓位：撤退→0%，中性→50%，弱信号→80%，强信号→100%。同时解决两个问题：
1. 回测起点应空仓等强信号（避免2018年半仓吃熊市）
2. 减少信号边界震荡导致的频繁调仓（确认机制）

### 信号层级定义

| 层级 | 条件 | 目标仓位 |
|------|------|:--:|
| 撤退 | ratio_d ≥ 92% OR ratio_s ≥ 92% | 0% |
| 中性 | 不满足其他条件 | 50% |
| 弱信号 | ratio_d ≤ 15% OR ratio_s ≤ 40%（仅单倍） | 80% |
| 强信号 | ratio_d ≤ 15% AND ratio_s ≤ 40%（双倍+单倍同时） | 100% |

附加规则：撤退清仓后，仅强信号可首次重入（was_cleared 锁）。

### 确认机制（CONFIRM_DAYS = 5）

信号权重需连续5个交易日保持不变，才触发实际调仓。避免阈值边界震荡导致频繁买卖。

### 50/50 沪深300+创业板回测（2018-01 ~ 2026-05）

| 指标 | 基线(全仓进出) | 动态分层 v2 | 买入持有 |
|------|:--:|:--:|:--:|
| 总收益率 | **146.4%** | 115.8% | 73.5% |
| Sharpe | 0.65 | 0.64 | 0.40 |
| 最大回撤 | -38.7% | **-35.8%** | -50.3% |
| 交易次数 | 3 | 19 | — |

### 调仓日志

```
2019-01-18  FULL      ← 起点空仓，等5天确认后首次满仓入场
2019-02-28  NEUTRAL   ← 信号减弱
2019-05 ~ 09  WEAK↔NEUTRAL
2020-02-07  FULL
2021-01-07  CLEAR     ← 撤退触发
2022-03-31  FULL      ← 仅强信号重入
2022-06 ~ 2023-05  WEAK↔FULL
2024-10-15  WEAK      ← 9月行情后半段
2025-08     WEAK→NEUTRAL  ← 当前
```

### 结论

1. 动态分层比基线少 30pp 收益，主要原因是 CONFIRM_DAYS=5 的确认延迟导致调仓慢半拍
2. 确认机制有效：交易次数从 61 降到 19（-69%），减少边界震荡损耗
3. 回撤改善：-35.8% vs -38.7%，增量不大
4. **暂不接受动态分层**：基线 3 次交易已够低频，无需确认机制。分层在有频繁调仓的全量化策略上更有价值
