## 二、目标

从 T005 的教训出发：**同一种指标的不同周期（ROC(5/15/22)）投票失败了，因为因子高度共线。** 换一个思路——用**不同类型的动量指标**来投票（ROC + RSI + MACD）。

核心假设：当多种不同维度的动量指标同时指向同一方向时，信号的可信度更高；当它们分歧时，应该观望。

## 三、讨论共识（2026-05-05 确认）

| 议题 | 结论 |
|------|------|
| **投票方案顺序** | A（简单多数）→ B（加权投票）→ C（连续融合）→ D（严格AND），按序推进 |
| **首批指标** | ROC + RSI + MACD，3个。验证有效后再考虑扩展 |

| **优先级** | 先深挖动量策略，暂不分散到其他策略类型 |

## 四、实施计划

### Step 1：新增 RSIIndicator 和 MACDIndicator

在 `indicators/technical.py` 中新增两个指标类。

#### RSIIndicator（相对强弱指标，Wilder's RSI）

```
RSI = 100 - 100/(1 + RS)
RS = AvgGain(N) / AvgLoss(N)
AvgGain = EMA( max(ΔC, 0), N, alpha=1/N )    # Wilder 平滑
AvgLoss = EMA( max(-ΔC, 0), N, alpha=1/N )
```

参数：`n=14`，输出 `rsi` 列（0~100）。

#### MACDIndicator（指数平滑异同移动平均线）

```
EMA12 = EMA(C, 12)
EMA26 = EMA(C, 26)
DIF = EMA12 - EMA26
DEA = EMA(DIF, 9)
MACD柱 = 2 × (DIF - DEA)
```

参数：`fast=12, slow=26, signal=9`，输出 `dif`, `dea`, `macd_bar` 列。

**验收**：合成数据验证——RSI 在 0~100 之间，MACD DIF/DEA 在趋势行情中金叉/死叉位置正确。

### Step 3：实现投票决策流

在 `strategies/roc_momentum.py` 中新增 `_produce_voting_decisions()` 方法。

#### 3.1 信号离散化逻辑

每个指标在每个交易日产出三个信号之一：看多(+1) / 看空(-1) / 中性(0)：

| 指标 | 看多(+1) | 看空(-1) | 中性(0) |
|------|---------|---------|---------|
| ROC | ROC >= buy_roc_edge | ROC <= sell_roc_edge | 中间区间 |
| RSI | RSI >= rsi_bull_threshold | RSI <= rsi_bear_threshold | 中间区间 |
| MACD | DIF > DEA 且 macd_bar > 0（金叉状态） | DIF < DEA 且 macd_bar < 0（死叉状态） | DIF与DEA纠缠 |

#### 3.2 四种投票方法

**方法A — majority（简单多数）**：

```
净票数 = 看多票数 - 看空票数
if 净票数 >= voting_threshold_buy: 买入方向
if 净票数 <= voting_threshold_sell: 卖出方向
else: 观望
```

**方法B — weighted（加权投票）**：

```
净票数 = Σ(w_i × 信号_i)    # 信号为 +1/0/-1
阈值同方法A
权重由 indicator_weights 配置，未配置时等权
```

**方法C — continuous（连续融合）**：

```
各指标原始值 Z-score 归一化到 [-1, 1] 后加权求和
if 融合值 >= 买入阈值: 买入
if 融合值 <= 卖出阈值: 卖出
```

**方法D — strict（严格AND）**：

```
只有全部 N 个指标都看多，才买入
只要任一指标看空，就卖出（退出）
```

#### 3.3 三种集成方式

**方式1 — filter（保守）**：
- 排名仍用 ROC 单因子排序
- 买入需同时满足"ROC 条件通过" + "投票通过（净票数 >= 阈值）"
- 卖出逻辑不变（不在 TOP_K → 挤压卖出，或满足卖出条件 → 卖出）

**方式2 — boost（激进加分）**：
- 在 ROC 排名权重上叠加投票分数
- 综合得分 = ROC原始权重 + voting_boost × 净票数
- 按综合得分排序取 TOP_K

**方式3 — direction（仅方向）**：
- 排名仍用 ROC
- 投票只影响方向：投票通过 → 允许买入；投票不通过 → 强制 hold/exit
- 与 filter 的区别：filter 可能产生"买不够 K 个"，direction 在投票不通过时可能产生 exit

#### 3.4 NaN/异常值处理

- 如果某个指标的值为 NaN → 该指标本轮不参与投票（视为中性 0）
- 如果某标的超过半数指标为 NaN → 跳过该标的

**验收**：合成数据测试 —— 3看多 → 买入；2看多1中性 → 观望（阈值=2时）；1看多2看空 → 卖出。NaN 指标正确跳过。

## 五、验收结果

| **MACD_DIF** | 0.0048 | 0.0294 | 0.16 | 60% | ❌ |

### 5.3 分析解读

**V1（简单多数投票）与 Baseline 完全一致的原因**：

当 ROC(22) 足够高到触发买入（>= 20）时，RSI 和 MACD 几乎总是同时看多。三个因子高度共线（0.83-0.87），所以投票从未过滤掉任何 ROC 信号。投票 == 不投票。

**V2-B-filter（0笔交易）的原因**：

权重配置 RSI(0.5) > ROC(0.3) 导致 RSI 权重过高。在 filter 模式下，RSI 看多阈值 55 意味着即使 ROC 很强，只要 RSI 不够强，净票数就达不到阈值。过于保守 → 零交易。

**V3-C（连续融合）下降 23.7pp**：

Z-score 归一化抹平了 ROC 的"极端信号"能力。在截面归一化后，高 ROC 的标的在其他维度（RSI/MACD）上未必同样突出，融合分数反而降低了排序效果。

**V4-D（严格AND）下降 36pp**：

要求三指标全票通过 → 信号极稀疏。虽然交易次数增加了（7次），但胜率仅 28.6% → 频繁进出被佣金和滑点吃掉。

### 5.4 结论

**多指标投票增强不优于 ROC 单因子。**

- ROC + RSI + MACD 的因子值相关性 > 0.83，投票并未提供增量信息
- Baseline 的 Sharpe=1.37 已经是这批标的上动量策略的很强表现
- 所有投票变体要么与 Baseline 持平，要么显著更差

**但有一个重要副产物**：RSI(14) 的 **ICIR=0.83 远优于 ROC(22) 的 0.36**。这意味着：
- RSI 作为因子有更强的预测力
- 如果单独用 RSI 替代 ROC 来构建排名和信号，可能获得更好效果
- 这不是"投票"问题，而是"选哪个因子做主力"的问题

**建议行动**：
- 不要在 live 配置中启用 `voting_enabled`
- 保持 ROC(22) + inv_vol 作为当前配置
- 将**【T014 RSI 单因子替代 ROC 探索】**作为新的研究方向

## 八、备注

- **因子共线性是投票无效的根本原因**：ROC、RSI、MACD 虽然数学公式不同，但都从同一价格序列派生。在趋势市场中三者几乎总是同向
- **T005 + T012 = 动量投票战线的完整结论**：
  - 同质多周期投票（T005）：失败，因子共线性 → 负收益
  - 异质多指标投票（T012）：无效，因子共线性 → 与 Baseline 持平或更差
  - 最终结论：**在动量轮动策略中，单因子 ROC 已经足够。多因子投票/共识不会带来增量收益。**
- **RSI(14) 的高 ICIR 值得单独探索**：这可能是 T012 最有价值的发现
