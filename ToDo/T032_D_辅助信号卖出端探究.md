## 〇、动机

T028 通过 1444 个买入信号的归因分析，发现量价、波动率、MACD 背离三大辅助信号能有效区分 ROC 买入信号的质量。这些信号当前仅应用在**买入端**（`_evaluate` 的 `buy_ok` 分支），但逻辑天然对偶——**能预测"买入后会不会盈利"的信号，同样能预测"盈利会不会结束"**。

当前 ROC 策略的卖出端完全依赖价格动量自身：
- ROC < 3.0（趋势消退）
- MAROC < 0（动能转负）
- 高水位/成本止损（回撤保护）

没有任何外部确认信号。T028 提供了"趋势何时衰竭"的统计证据，值得做一次针对性的卖出端探究。

## 一、四个卖出信号的设计依据

### 1.1 放量卖出（volume_sell）

| 维度 | 内容 |
|------|------|
| **T028 买入端依据** | 放量买入胜率 61.5%，缩量 68.5%。放量 = 追涨盘涌入 = 后市空间有限 |
| **→ 卖出端反转逻辑** | 持仓中突然放量到正常水平的 1.5 倍以上 → 天量天价，趋势可能短期见顶 |
| **卖出规则** | `vol / MA(vol, 20) > 1.5` → 提前卖出（即使 ROC 尚未跌破阈值） |
| **与买入端关系** | 互为对偶：买入端过滤放量（别追涨），卖出端检测放量（该跑了） |
| **风险** | 放量也可能是趋势加速的信号——T028 数据中放量组平均持有 23 天，盈亏 +4.6%，并非全亏。需回测确认净效果 |

> 注：T028 的放量统计是针对买入信号触发时的成交量，持有期内的放量未必遵循同样的胜率规律。需要在持有期逐日监控 volume_ratio。

### 1.2 ATR 扩张卖出（atr_expansion_sell）

| 维度 | 内容 |
|------|------|
| **T028 买入端依据** | 高波买入胜率 66%（优于正常波 48.5%），但此统计针对买入时点 |
| **→ 卖出端反转逻辑** | 持有期中 ATR 突然扩张 = 市场进入极端波动，ROC 动量信号的可靠性急剧下降。T033 已验证 ATR 扩张对买入的阻断有效性，卖出端是自然延伸 |
| **卖出规则** | `ATR(20) > 1.5 × ATR(200)` → 强制卖出（阈值比买入端 1.3 更宽松） |
| **与买入端关系** | 买入端：ATR 扩张时不买（已经在恐慌中）；卖出端：ATR 扩张时跑（恐慌正在发生） |
| **风险** | 宽松阈值（1.5 vs 1.3）是为了避免噪音退出，但可能仍然太敏感 |

### 1.3 MACD 顶背离卖出（macd_divergence_sell）

| 维度 | 内容 |
|------|------|
| **T028 买入端依据** | 顶背离买入胜率 44% vs 无背离 65%（降幅 20pp） |
| **→ 卖出端反转逻辑** | 持仓中收盘价创 lookback 日新高，但 DIF 未同步创新高 → 趋势衰竭，即使 ROC 仍然很高也应减仓或退出 |
| **卖出规则** | `close 创 20 日新高 AND DIF 未创 20 日新高` → 提前卖出 |
| **与买入端关系** | 复用 `_check_divergence` 函数，检测逻辑完全相同 |
| **风险** | T028 中背离信号仅 77/1449 个买入信号（5.3%），持有期内出现的频率可能更低 |

### 1.4 【添头】RSI>80 止盈（rsi_sell）

| 维度 | 内容 |
|------|------|
| **T028 买入端依据** | RSI≥80 的买入信号仍有 59% 胜率 +5.6% 盈亏——超买不是绝对禁止。但早期组 RSI≥80 盈亏 +8.2%，中期组降到 +3.6%，说明超买越久越危险 |
| **→ 卖出端逻辑** | 持仓中 RSI 突破 80 → 短期极度超买，止盈锁定利润。这不是"截断利润"——止损留给高水位/成本止损，这个是锦上添花 |
| **卖出规则** | `RSI(14) > 80` → 提前止盈卖出 |
| **与前三信号的层级** | **添头**：不单独回测、不参与组合对比。三个主力信号定稿后，在全套配置上随手加一个开关跑一次看效果 |
| **风险** | 趋势极强的标的上 RSI>80 后可能继续涨（T028 中 RSI≥80 盈亏 +5.6%）——所以是添头不是主力 |

## 二、技术方案

### 2.1 实现路径

在 `_evaluate` 的卖出分支中，于现有 `roc_val < sell_roc_edge` 和 `maroc < sell_ma_roc_edge` **之后**，插入三种新卖出条件：

```python
# ---- 卖出判断 ----
if code in positions:
    # 现有：ROC/MAROC 基础退出
    if roc_val < self._config.sell_roc_edge:    return 'exit', ...
    if maroc_val < self._config.sell_ma_roc_edge: return 'exit', ...

    # === T032 新增：辅助信号卖出 ===

    # 1. 放量卖出
    if self._config.volume_sell_enabled:
        vol_ratio = ind.get('hold_vol_ratio', -1.0)
        if vol_ratio >= self._config.volume_sell_spike_ratio:
            return 'exit', 0.0, f"放量卖出: vol_ratio={vol_ratio:.1f}"

    # 2. ATR 扩张卖出
    if self._config.atr_expansion_sell_enabled:
        expansion = ind.get('hold_atr_expansion')
        if expansion is True:
            return 'exit', 0.0, "ATR扩张卖出: ATR(20)>1.5×ATR(200)"

    # 3. MACD 顶背离卖出
    if self._config.macd_divergence_sell_enabled:
        divergence = ind.get('hold_macd_divergence')
        if divergence is True:
            return 'exit', 0.0, "MACD顶背离卖出: 价新高DIF未确认"

    # 4. [添头] RSI>80 止盈
    if self._config.rsi_sell_enabled:
        rsi_val = ind.get('hold_rsi')
        if rsi_val is not None and rsi_val > 80:
            return 'exit', 0.0, f"RSI>80止盈: RSI={rsi_val:.1f}"
```

### 2.2 关键差异：持有期监控 vs 买入时点检测

买入端只检测"买入时那一瞬间"的信号。卖出端需要在**整个持有期**监控信号。这意味着 `indicator_data` 中不能只用 `latest` 的值——需要为持仓中的标的额外计算持有期专用指标。

具体做法：在 `_produce_singlefactor_decisions` 中，对 `code in positions` 的标的，在现有 `indicator_data[code]` 基础上补充以下字段：

```python
if code in positions and (
    self._config.volume_sell_enabled or
    self._config.atr_expansion_sell_enabled or
    self._config.macd_divergence_sell_enabled or
    self._config.rsi_sell_enabled
):
    # 放量比：当日成交量 / 前20日均量
    if self._config.volume_sell_enabled and 'vol' in df.columns:
        vol_arr = df['vol'].values.astype(float)
        indicator_data[code]['hold_vol_ratio'] = vol_arr[-1] / vol_arr[-21:].mean()

    # ATR 扩张：持仓中实时计算
    if self._config.atr_expansion_sell_enabled:
        indicator_data[code]['hold_atr_expansion'] = self._check_atr_expansion(df, ratio=1.5)

    # MACD 顶背离：复用买入端函数
    if self._config.macd_divergence_sell_enabled:
        indicator_data[code]['hold_macd_divergence'] = self._check_divergence(df)

    # RSI>80：从现有 indicator_data 直接取（已在买入端计算）
    if self._config.rsi_sell_enabled:
        indicator_data[code]['hold_rsi'] = indicator_data[code].get('rsi')
```

### 2.3 新增 Config 字段

在 [roc_config.py](file:///e:/JuJu/TraeProjects/量化工程/quantforge/strategies/_configs/roc_config.py) 的 `#### →信号过滤←` 区块新增：

```python
# === T032: 辅助信号卖出（探究性） ===
volume_sell_enabled: bool = False              # 成交量放量卖出：持仓中 vol_ratio>=阈值 → exit
volume_sell_spike_ratio: float = 1.5           # 放量阈值
atr_expansion_sell_enabled: bool = False       # ATR扩张卖出：持仓中ATR(20)>1.5×ATR(200) → exit
macd_divergence_sell_enabled: bool = False     # MACD顶背离卖出：持仓中价新高DIF未确认 → exit
rsi_sell_enabled: bool = False                 # [添头] RSI>80止盈：持仓中RSI>80 → 止盈退出
```

> **设计原则**：每个卖出开关独立于买入开关。`macd_divergence_sell_enabled` 和 `macd_divergence_filter_enabled` 互不依赖，可单独测试。

### 2.4 `_check_atr_expansion` 签名扩展

当前 `_check_atr_expansion(df)` 硬编码 1.3 倍阈值。扩展为：

```python
def _check_atr_expansion(self, df, ratio=1.3):
```

买入端调用 `_check_atr_expansion(df)`（默认 1.3），卖出端调用 `_check_atr_expansion(df, ratio=1.5)`（更宽松）。

## 五、回测结果（2018-2026 全周期，tech_growth 33池）

### 8组配置汇总

| 配置 | 年化 | 总收益 | Sharpe | 回撤 | 交易 | 被打断 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| **baseline** | **17.1%** | **175.2%** | **1.32** | 14.2% | 19 | — |
| vol_sell_1.5 | 4.6% | 33.6% | 0.68 | 10.5% | 19 | 4笔 |
| atr_sell_1.5 | 17.1% | 175.2% | 1.32 | 14.2% | 19 | 0笔 |
| macd_div_sell | 9.7% | 81.2% | 1.32 | 5.2% | 19 | 11笔 |
| rsi_sell_80 [添头] | 11.8% | 104.8% | 1.24 | 9.1% | 19 | 2笔 |
| vol+atr | 4.6% | 33.6% | 0.68 | 10.5% | 19 | 4笔 |
| vol+atr+macd | 6.5% | 50.3% | 1.06 | 6.6% | 19 | 11笔 |
| full+rsi | 6.5% | 50.2% | 1.07 | 6.6% | 19 | 11笔 |

### 逐信号分析

**1. 放量卖出（vol_sell）— 最差**
- 4笔触发，年化从17.1%降到4.6%，Sharpe从1.32降到0.68
- 被打断交易总盈亏仅+116，但更重要的是：**放量在ROC动量持有期中太频繁，切掉了大趋势中的正常放量加速**
- 结论：T028买入端的"放量不可靠"不等于"放量该卖"。强趋势中的放量是趋势确认而非终结信号。

**2. ATR扩张卖出（atr_sell）— 零信号**
- 8年XX只标的的持有期间，从未触发ATR(20)>1.5×ATR(200)
- 原因：ROC买入后持有期通常20~40天，这段时间内ATR200基本走平；ATR20偶尔扩张但难超1.5倍
- 结论：阈值合适但信号密度为零，无法实际使用

**3. MACD顶背离卖出（macd_div_sell）— 最强但不值得**
- 11笔触发，年化9.7% vs baseline 17.1%。回撤5.2% vs 14.2%（降幅显著）
- 但年化减半的代价太大——被打断的11笔交易在baseline中共盈利31,815，而"提前卖出"只锁定了该利润的一部分
- 结论：MACD顶背离确实预示趋势放缓，但过早退出错过了余下的盈利

**4. RSI>80止盈（rsi_sell）— 温和但无性价比**
- 仅2笔触发——RSI>80在33只科技成长ETF的强趋势中极少出现
- 年化从17.1%降到11.8%，两笔触发各切掉了一个中等趋势
- 结论：信号密度太低，且触发的时机不在趋势终点

### 最终判定

> **四个辅助卖出信号均不能改善基线。Go/No-Go = No-Go。**
>
> T028的买入端发现（放量/缩量/波动率/背离影响胜率）**不能直接对偶到卖出端**。核心原因：
> 1. 持有期中的量/波/背离信号与买入时点的信号特性完全不同
> 2. 买入时点是截面的（在多个标的中选最优），卖出时点是时序的（在单个持仓中判断退出时机）
> 3. 现有的ROC<3.0+MAROC<0+高水止损已经足够有效——辅助信号在卖出端是画蛇添足
>
> **建议**：四个开关保留在代码中供后续研究使用，但默认关闭，不进实盘。

## 七、设计原则（探究阶段约束）

1. **不改变默认行为**：所有新开关 `default=False`
2. **买入/卖出开关独立**：`volume_sell_enabled` 不依赖 `volume_filter_enabled`
3. **每个信号可独立测试**：AB 回测脚本支持 `--enable volume_sell` 等单参数
4. **"被打断交易对比"是决定性证据**：如果辅助卖出提前退出了一个大赢家，这个信号就不适合
