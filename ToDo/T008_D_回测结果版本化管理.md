## 二、目标

将每次回测结果与 Git 提交 SHA 绑定，实现：
1. 每次回测自动记录生成结果的代码版本
2. 可追溯：某次回测由哪次 commit 的代码产生
3. 可对比：两个不同版本的代码，回测结果差异是什么

## 三、实施计划

### Step 1：在 BacktestAnalyzer 中注入 Git 信息

```python
# research/backtest_analyzer.py

import subprocess

def _get_git_info(repo_path: str) -> dict:
    """获取当前 Git 仓库信息"""
    try:
        sha = subprocess.check_output(
            ['git', 'rev-parse', '--short', 'HEAD'],
            cwd=repo_path, text=True
        ).strip()
        branch = subprocess.check_output(
            ['git', 'rev-parse', '--abbrev-ref', 'HEAD'],
            cwd=repo_path, text=True
        ).strip()
        dirty = subprocess.check_output(
            ['git', 'status', '--porcelain'],
            cwd=repo_path, text=True
        ).strip() != ''
        return {'sha': sha, 'branch': branch, 'dirty': dirty}
    except Exception:
        return {'sha': 'unknown', 'branch': 'unknown', 'dirty': False}
```

## 六、备注

- **GitPython 依赖**：当前项目中已有 `gitpython` 依赖（`git_pull.py` 使用了 `from git import Repo`）。可以用 `gitpython` 的 API 替代 `subprocess` 调用 Git 命令，更稳定可靠
- **index.json 的维护**：不要每次回测都重写整个 index。使用追加模式——读取现有 index，追加新记录，写回。长期积累可能变大，后续可考虑只保留最近 N 条
- **对比工具的范围**：当前只对比基础指标（return/sharpe/drawdown）。后续可扩展到交易记录差异、持仓重叠度等
- **与 T006/T007 的关系**：T006 的验证管道报告同样应包含 Git SHA。T007 的配置预设名也应记录在 index 中
