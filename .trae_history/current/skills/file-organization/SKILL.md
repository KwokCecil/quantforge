# 文件归档规范

> **自动触发**：新建文件、移动文件、清理旧文件时自动参考。

---

## 一、目录用途一览

| 目录 | 用途 | 生命周期 |
|------|------|---------|
| `ToDo/` | 施工文档（T编号 .md） | 永久保留，不移动 |
| `指导文档/` | 规格书、复盘、使用指南等 | **永久保留，不可删** |
| `临时文件/` | 分析中间产物、暂存旧脚本 | 可清理 |
| `results/` | 回测最终产出目录 | 保留最近 10 次，旧可删 |
| `tests/` | 测试脚本 | 永久保留 |
| **根目录** | 入口脚本、运维脚本、运行时状态 | 随项目维护 |

---

## 二、文件归属决策树

```
新文件放哪？
    │
    ├─ 是子任务施工文档？                              → ToDo/
    ├─ 是长期参考文档（规格书/复盘/使用指南/分析）？     → 指导文档/
    ├─ 是测试/验证脚本？                               → tests/
    ├─ 是回测引擎产出的最终结果目录？                    → results/{run_id}/
    ├─ 是分析中间产物（CSV/JSON/阶段性报告）？           → 临时文件/
    ├─ 是被替代的旧脚本，拿不准能不能删？               → 临时文件/（暂存）
    ├─ 是程序主入口（main_*.py）？                      → 根目录
    ├─ 是运维脚本（*.bat / *_pull.py / remote_cmd.*）？  → 根目录
    ├─ 是运行时状态文件（position.json / 日志）？       → 根目录
    └─ 是依赖/环境配置（requirements.txt / .gitignore）？ → 根目录
```

## 三、指导文档命名规则

格式：`{阶段号}.{序号}_{名称}.md`

| 阶段号 | 含义 | 示例 |
|--------|------|------|
| 1 | 奠基规格 | `1.01_工程规格书.md` |
| 2 | 规划蓝图 | `2.01_施工文档总纲.md` |
| 3 | 评估诊断 | `3.01_项目现状与展望.md` |
| 4 | 实施复盘 | `4.01_子任务实施复盘.md` |
| 5 | 使用参考 | `5.01_使用指南.md` |

**扩编**：取该阶段当前最大序号 +1，不同阶段独立编号，互不冲突。

---

## 四、清理规则

- **`临时文件/`**：新分析运行会自动覆盖同名旧文件。确认无用的直接删除。
- **`results/`**：保留最近 10 次，更早的可删。
- **`指导文档/`**：**不可删**。旧版也保留作历史参照。
- **`ToDo/`**：**不可删**。已完成的施工文档是项目"决策日志"。
- **根目录旧脚本**：已被模块替代的删，不确定的移 `临时文件/` 暂存。

---

## 五、文档维护原则（分层维护）

指导文档和施工文档遵循**两层维护策略**，兼顾"随时可用"和"可追溯演进"：

### 第一层：最新说明（必须保持最新）

| 文档类型 | 示例 | 过时后果 |
|----------|------|----------|
| 使用指南（怎么用） | `5.01_使用指南.md` | 新人照着做会出错，误人子弟 |
| 成果总结（什么结论） | `5.12_现阶段成果总结.md` | 引用的参数/绩效数据无效 |
| 参数/稳健性验证 | `5.13_全周期参数稳健性重验证.md` | 决策依据错误 |

**维护要求**：每次重大变更后同步更新。参数变了、入口改了、结论更新了，**必须**更新对应文档。

### 第二层：历史档案（保留原貌，加标注指向）

| 文档类型 | 示例 | 处理方式 |
|----------|------|----------|
| 施工文档（ToDo/） | `T025_D_...` | 完成后不改内容，是"决策日志" |
| 分析报告（3.x系列） | `3.01_项目现状与展望.md` | 顶部加 ⚠️ 归档标注，指向当前权威文档 |
| 方法教程（1.x系列） | `1.04_IC分析与参数优化.md` | 方法论保留，数据过时处加来源说明 |

**处理方式**：不修改原文。在顶部醒目标注：
```markdown
> ⚠️ **本文档已归档（YYYY-MM-DD）**
> ...说明过时原因...
> **当前权威文档**：[xxx.md](./xxx.md)
> 本文档保留作为历史演进记录。
```

### 归档触发条件

满足以下任一条件即触发归档：
- 引用的模块路径/文件名已变更
- 参数推荐值已被后续验证推翻
- 数据来自已被替代的回测引擎（如 v2→v3）
- 文件结构已大幅演进（新增/删除目录）
