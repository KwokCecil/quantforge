## 一、为什么去世坤

### 1.1 你现在的水平

你已经独立完成了一个完整的量化策略系统：
- **知道**：什么是因子、什么是 IC/ICIR、怎么做全周期回测、怎么验证参数稳健性
- **不知道**：一套因子是否在说同一件事（共线性）、怎么剥离行业/市值 β、怎么判断两个因子的边际贡献、因子衰减怎么监控

这些是你 ETF 阶段天然不会遇到的问题——因为 ETF 只有几十个标的，行业/市值结构相对固定。但**个股阶段每个问题都躲不开**。

### 1.2 世坤 BRAIN 恰好补你缺的

世坤不教你怎么写策略，它教你**什么是好的因子**：

| 你现在的认识 | 世坤会教的 |
|-------------|-----------|
| IC > 0 就是好因子 | IC ≠ 预测力。IC 可能来自行业暴露而不是因子本身 |
| 因子相关就换一个 | 因子正交化：怎么"萃取"不相关的部分 |
| 用全周期回测判断好坏 | 按时间分训练集/测试集，避免前视偏差 |
| 一篮子 ETF 买就完了 | 市场中性化：long-short 权重向量，零 β 暴露 |
| 策略开了 17 个开关 | 叠因子要证明边际提升，不能靠"全开" |

### 1.3 世坤 vs QuantForge 的分工

| | 世坤 BRAIN | QuantForge |
|---|---|---|
| **做** | 发现、检验、提交 α 因子 | 把因子组合成可实盘执行的策略 |
| **数据** | 全球股票，平台提供 | A 股 ETF/个股，自行接入 |
| **产出** | 单个 α 的 Sharpe / Turnover / Fitness | 完整策略的净值 / 回撤 / 实盘交易 |
| **你的角色** | 因子研究员 | 策略设计师 + 系统工程师 |

世坤 = 因子工厂。QuantForge = 策略装配线。先去工厂学会怎么造好零件，回来才能造好产品。

## 二、世坤学习路线图

### Phase 1：平台入门（第 1-3 天）

**目标**：能用 BRAIN API 提交一个最简单的因子，看懂基础指标。

#### 必学

1. **注册 + 平台熟悉**
   - 注册 IRC 账号：worldquantbrain.com
   - 了解 BRAIN 平台 5 个核心页面：Simulator / Data / Alphas / Leaderboard / Learn
   - 阅读官方文档：[How BRAIN Platform Works](https://platform.worldquantbrain.com/learn/documentation/create-alphas/how-brain-platform-works)

2. **写第一个 Alpha**
   ```python
   # 最简单的动量 alpha
   alpha = -close  # 负的收盘价：越便宜越买
   ```
   - 理解为什么 `-close` 能有正收益（价值因子逻辑）
   - 看回测结果：Sharpe / Turnover / Fitness / Returns / Drawdown / Margin

3. **看懂关键指标**

   | 指标 | 含义 | QuantForge 对应 |
   |------|------|----------------|
   | **Sharpe** | 风险调整后收益 | `analysis['sharpe_ratio']` |
   | **Turnover** | 换手率，越低越好 | `trades.length / trading_days` |
   | **Fitness** | Sharpe × 稳定性 × 与已有 α 的相关性惩罚 | 无直接对应（重要新概念） |
   | **Returns** | 年化收益 | `analysis['annual_return']` |
   | **Drawdown** | 最大回撤 | `analysis['max_drawdown']` |
   | **Margin** | 收益风险敞口（bps） | 无直接对应 |
   | **Long Count / Short Count** | 做多/做空标的数 | QuantForge 目前只做多 |

4. **理解 Neutralization（中性化）**

   这是世坤最重要的概念，也是你 ETF 阶段完全没碰过的：

   ```
   原始因子: close（便宜程度）→ 天然偏银行股（因为银行股就是便宜）
   Market 中性化后: 剥离了"整个市场平均便宜程度"，只留下"相对便宜"的那部分
   Sector 中性化后: 剥离了"行业内平均便宜程度"，只留下"行业内相对便宜"的那部分
   ```

   这就是你 ETF 阶段 IC 分析和策略收益之间那层"对不上"的关键原因。

#### 产出
- 成功提交 ≥ 1 个 alpha
- 理解 Neutralization 的数学原理（去均值 → 零和权重向量）
- 能解释 Fitness 比 Sharpe 多衡量了什么

### Phase 2：理解因子设计方法论（第 4-7 天）

**目标**：不再靠"猜"，而是有方法论地设计因子。

#### 必学

1. **数据字段分类**

   世坤把数据分了几十个数据集，你只需要先聚焦 3 类：

   | 类别 | 数据集 | 你已会的 | 要新学的 |
   |------|--------|---------|---------|
   | Price-Volume | `price_volume_1` / `model_77` | ROC/MA/RSI/MACD/ATR | `ts_rank`、`ts_delta`、截面 vs 时序 |
   | Fundamentals | `fundamental6` | 没有 | PE/PB/ROE/负债率、财报频率、前视偏差防护 |
   | Model（参考） | `model_51` / `model_53` | 没有 | 借鉴已有成熟因子的结构，理解"好因子长什么样" |

2. **Operator（算子）分类**

   世坤提供了固定的算子库，你不能写任意 Python 代码，只能用算子组合——这其实是在训练你的"因子思维"：

   | 类型 | 算子 | 作用 |
   |------|------|------|
   | 截面标准化 | `rank`、`zscore`、`winsorize` | 让不同量纲的数据可比 |
   | 时序 | `ts_rank`、`ts_delta`、`ts_mean`、`ts_zscore` | 捕捉趋势/变化 |
   | 逻辑 | `filter`、`sigmoid` | 分段/非线性 |
   | 组合 | `+`、`-`、`*`、`/`、`group` | 多字段融合 |

   这正好对应你在 `roc_momentum.py` 里用 pandas 手写的 `close / close.shift(22) - 1`、`rolling(...).mean()` 等——只是世坤把它抽象成了一套标准语言。

3. **因子设计三步法**

   ```
   ① 选数据字段 → ② 套算子 → ③ 看回测（Sharpe > 1.0, Fitness > 0.5 初步合格）
         ↓ 如果不通过
   ④ 调窗口（5天 → 20天?）→ ⑤ 调方向（正号 → 负号?）→ ⑥ 换数据字段
   ```

4. **两个关键概念**

   - **数据延迟（Delay）**：财务数据发布有滞后，回测必须模拟"当时能拿到的数据"。你 ETF 阶段从来没遇到这个问题（日 K 数据当天就有）——个股的 PE/ROE 财报数据不 delay 的话会有严重前视偏差。
   - **覆盖率（Coverage）**：一个因子在多少只股票上能计算出来。覆盖率 < 30% 的因子在 BRAIN 上会被降分——对应到 QuantForge 就是你写策略时标的数量不能太少。

#### 产出
- 理解至少 3 类数据字段的特点和适用场景
- 能独立设计"选数据 → 套算子 → 看回测"的闭环
- 提交 ≥ 3 个 Sharpe > 1.0 的 alpha

### Phase 3：进阶——正交化与中性化（第 8-14 天）

**目标**：这是 QuantForge 下一阶段最需要的能力——判断你的 5 个因子是不是在说同一件事。

#### 必学

1. **中性化的四层含义**

   | 中性化 | 剥离什么 | 为什么需要 |
   |--------|---------|-----------|
   | Market | 市场整体涨跌 | 不让"大盘好"伪装成"因子好" |
   | Sector | 行业差异 | 不让"行业热"伪装成"选股强" |
   | Subindustry | 子行业差异 | 更细粒度的行业剥离 |
   | None | 什么都不剥 | 看原始因子有多大的行业/市值暴露 |

2. **因子正交化（Orthogonality）**

   世坤有一个"Alpha 相关性矩阵"概念：你提交的新 alpha 如果和历史已有 alpha 相关性 > 0.7，会被显著降分。量化的不是"这个 alpha 好不好"，而是"这个 alpha 有没有告诉我我不知道的事"。

   这对应到你 ETF 阶段最惨痛的教训：**ROC、RSI、MACD 三个 IC 都为正，但因为相关性 > 0.83，投票和三叉戟全部无效**。如果你当时有正交化意识，就不会在三个共线因子上花那么多时间。

3. **Fitness 的数学含义**

   ```
   Fitness = Sharpe × Stability × (1 - MaxCorrelation)
                     ↑           ↑            ↑
                  年化收益    收益稳定性    与已有alpha的相关性
   ```

   - 一个 Sharpe 3.0 但与已有 alpha 高度相关（r>0.9）的新 alpha → Fitness 极低
   - 一个 Sharpe 1.0 但与已有 alpha 完全独立（r<0.2）的新 alpha → Fitness 可接受

   这是你 ETF 阶段完全缺失的维度——你只会看"这个开关开了收益涨多少"，但从来看"这个开关和其他开关是不是独立的"。

4. **因子衰减（Decay）**

   因子不是永动机。世坤教你用 decay 函数模拟"信号能持续多久"：
   - 日频因子可能衰减很快（1-3 天）
   - 月频因子衰减慢（1-3 个月）
   - 衰减速度决定你的调仓频率

   对应到 QuantForge：你现在的策略是固定每 2 周调仓，但实际上是"因子衰减速度 → 最优调仓频率"。这是一个可以优化的点。

#### 产出
- 能解释中性化如何改变一个因子的截面排名
- 能判断两个 alpha 是否冗余（靠相关性而不是凭感觉）
- 理解 Fitness 是你 ETF 阶段最缺的评估维度

## 四、回来后：QuantForge 二阶段升级路线

### 4.1 FactorLab 2.0（核心升级）

世坤回来后，FactorLab 从"单因子 IC 计算器"升级为"多因子研究平台"：

#### 新增能力

```python
class FactorLab:
    # === 现有能力（保留） ===
    @staticmethod
    def compute_ic(factor_values, forward_returns, method='spearman') -> dict:
        """IC/ICIR：单因子截面预测力"""  # 已有

    @staticmethod
    def compute_ic_matrix(factor_dict, forward_returns) -> pd.DataFrame:
        """多因子 IC 矩阵扫描"""  # 已有

    @staticmethod
    def layered_backtest(factor_values, forward_returns, n_groups=5) -> pd.DataFrame:
        """分层回测：因子分 5 组，看各组平均收益"""  # 已有

    # === 新增：中性化 ===
    @staticmethod
    def neutralize(factor_values: pd.Series,
                   industry: pd.Series,
                   market_cap: pd.Series) -> pd.Series:
        """行业+市值中性化。回归掉行业哑变量和 log(市值)，返回残差 = 纯 α"""

    # === 新增：正交化 ===
    @staticmethod
    def orthogonality_matrix(factor_dict: dict[str, pd.Series]) -> pd.DataFrame:
        """因子相关性矩阵。标记 r>0.7 的冗余对，建议只保留 IC 最高的"""

    # === 新增：Fitness ===
    @staticmethod
    def fitness(sharpe: float, ic_stability: float,
                max_correlation: float) -> float:
        """综合因子质量评估"""

    # === 新增：衰减分析 ===
    @staticmethod
    def compute_ic_decay(factor_values, forward_returns_multi_horizon) -> pd.DataFrame:
        """IC 衰减曲线：看因子预测力在 1d/3d/5d/10d/20d 如何衰减"""
```

#### 这对应你 ETF 阶段五个没解决的问题

| 问题 | FactorLab 2.0 怎么解决 |
|------|----------------------|
| 三个因子相关性 > 0.83 还在叠 | `orthogonality_matrix()` 直接标红 |
| 不知道超额是行业选得好还是因子强 | `neutralize()` 剥离行业暴露 |
| 只靠全周期回测判断好坏 | `fitness()` 三维评价（Sharpe+稳定+独立） |
| 固定两周调仓不知道对不对 | `compute_ic_decay()` 找到最优 rebalance |
| 财报数据的延迟问题 | `DataRequest(delay_days=45)` 模拟真实延迟 |

### 4.3 策略框架：ROC 动量在个股上

核心改动很小——因为你的架构已经是 Strategy → Decision → Resolver 三层解耦：

| **Decision** | `DecisionType.ROTATION` | 同上 |
| **Resolver** | `RankingResolver(top_k=5)` | 同上，但需要加涨跌停过滤 |
| **Executor** | `BacktestExecutor` | 同上，但 `_can_fill()` 要检查涨跌停 |

你不需要重写策略框架——只需要：

1. 换数据源（`AKShareFeed` 替代 `SinaFinanceFeed`）
2. 加过滤（ST、涨跌停、流动性、覆盖率）
3. 加中性化（`FactorLab.neutralize()`）
4. 重新跑 IC 矩阵 + Walk-Forward（新数据集上新参数）
