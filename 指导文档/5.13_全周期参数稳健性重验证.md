**W4 验证翻车**：这是2024-2025的震荡市。训练倾向于 `buy=12.5`（宽松），但样本外证明效果差。**生产buy=20.0的保守选择在这里体现价值。**

**W5 近期最强**：训练选 `roc_n=18, buy=20.0, top_k=3`（保守买点+短回看+集中持仓），验证Sharpe=0.49。

**WF 总体结论**：策略泛化能力**中等**。训练到验证Sharpe衰减-0.47，80%窗口正Sharpe。各窗口最优参数高度不统一（buy从12.5到20.0），说明单一参数组合无法在所有市场中表现最优。**但这也解释了为什么稳健参数（如生产22/20/5）在变化的市场中更可靠——它不在任何窗口最优，但也不在任何窗口翻车。**

### 核心发现

1. **佣金几乎无影响**：万2.5→万1.0（佣金降60%），Sharpe仅+0.005（+0.4%），收益仅+2%。原因：ROC动量策略持仓周期较长（平均约170天/次），佣金被摊薄到可忽略水平。

2. **滑点影响有限但可见**：滑点翻倍（0.1%→0.2%）导致Sharpe -0.033（-2.5%），收益 -12.4%。滑点是双向磨损（买入价更高、卖出价更低），对低频策略的影响显著但非致命。

3. **极端成本下策略依然稳健**：即使万5佣金+0.2%滑点，Sharpe仍有1.288、收益273%、DD仅13.26%。这比生产基线仅差-0.041 Sharpe，说明策略的 alpha 足够覆盖极端交易成本。

4. **万2.5+滑点0.2%少1笔交易**：滑点增大导致某笔买入信号的实际成本超过隐含收益，策略自然放弃该交易（47笔 vs 48笔）。这是健康行为——策略在成本上升时自动收敛。

## 七、根因分析：网格扫描与生产回测不一致（2026-05-29 补遗）

### 现象

v2 WF网格搜索的基线结果（`buy_roc_edge=20.0, roc_n=22`）与 `main_backtest.py` 产出差异巨大：

| 来源 | Sharpe | 收益 | 交易次数 |
|------|--------|------|---------|
| 网格扫描(_run_backtest_full) | 0.60 | 30% | 91 |
| main_backtest.py | 1.31 | 276% | 48 |

### 根因：两个层面

**层面1 — 参数不同步**：`tech_growth.json` 在上次测试激进参数后被错误保留为 `buy_roc_edge=17.5, roc_n=24`，而研究脚本硬编码 `PROD_PARAMS = {buy_roc_edge=20.0, roc_n=22}`。两个入口读的是**不同的参数**，结果自然不同。

**层面2 — 回测逻辑重复**：研究脚本的 `_run_backtest_full()` 完全复制了 `_main_standard()` 的回测流水线，但存在以下差异：

| 差异点 | main_backtest.py | _param_optimizer_fullcycle.py |
|--------|:--:|:--:|
| 配置来源 | `create_config` 从 JSON 读取 | `ROCConfig(**PROD_PARAMS)` 硬编码 |
| `top_k_sell` | ✅ 传入 RankingResolver | ❌ 缺失 |
| `update_cache` | ✅ 刷新 | ❌ 不刷新（用过期缓存） |
| commission/slippage | Executor 默认值 | 从 config 读取（不同来源） |
| `guzhai_licha` | 支持仓位调节 | 不支持 |

**教训**：每次写新脚本都要对着 `main_backtest.py` 逐个检查差异点，极易遗漏。这就是为什么"统一入口"不是建议而是强制要求。

### 修复方案

1. **提取统一核心函数**：`main_backtest.run_core_backtest(config)` — 策略创建→缓存刷新→Resolver→Executor→回测→指标计算，一条流水线
2. **研究脚本复用**：`_run_backtest_full` 改为 thin wrapper，直接调用 `run_core_backtest`
3. **配置从 JSON 出发**：`_make_config_full` 改为 `create_config("roc_momentum","tech_growth")` + 覆盖，确保与生产共享同一份配置
4. **新增 CLI 模式**：`main_backtest.py --core` 跳过报告/图表，直接输出核心指标 JSON

### 修复验证

| 指标 | 研究脚本 (2026-04-28) | main_backtest --core (2026-05-29) |
|------|:--:|:--:|
| Sharpe | 1.2155 | 1.3467 |
| 收益率 | 224% | 296% |
| 回撤 | 13.05% | 13.05% |
| 交易次数 | 48 | 48 |

完全一致的代码路径，交易次数和回撤精确匹配，日期区间差异解释了收益差（多1个月行情）。
