# tests / research / results 目录职责边界

> **创建日期**：2026-05-13
> **目的**：严格区分三个容易混淆的目录，防止分析脚本混入测试、测试脚本写入结果等混乱。
> **适用范围**：所有新建或迁移的文件。

---

## 一、三元区分

| | `tests/` | `research/` | `results/` |
|---|---|---|---|
| **一句话定义** | 验证正确性的自动化检查 | 探索性分析、实验、工具库 | 回测/扫描产出的数据 |
| **核心问题** | "这代码还对吗？" | "这样做好不好？" | "那次跑出了什么？" |
| **内容性质** | 断言驱动，输出 pass/fail | 计算驱动，输出图表/报告/发现 | 数据驱动，只读不执行 |
| **运行方式** | `python _test_xxx.py`（返回 0/1） | 手动执行，结果需人解读 | 不运行，只被读取 |
| **依赖外部** | / 单元+契约层不依赖 | 可以（需网络/缓存/文件） | — |

---

## 二、tests/ — 验证正确性

### 入目录标准（全部满足）

1. 运行时返回**显式 pass/fail**（exit code 0 = 通过，非 0 = 失败）
2. 使用 **assert** 或等价检查
3. 目的是验证 **代码行为是否符合预期**

### 不入 tests/

- 跑完只输出报告、没有断言的脚本 → `research/`
- 参数扫描工具（网格搜索最优参数）→ `research/`
- 一次性探索脚本（跑完就删的那种）→ `research/`

### 分层标记

每个测试文件第一行必须标注所属分层：

```
# @layer: unit       — 单函数输入→输出，合成数据，<1s
# @layer: contract   — 两个实现的行为一致性，合成数据，<5s
# @layer: integration — 策略+执行器完整链路，可能需网络/缓存
# @layer: e2e        — 全周期回测对比，数分钟
```

分层定义详见 `指导文档/3.10_单元测试制度_问题分析与系统方案.md`。

### 自动发现与运行

`tests/run_all_tests.py` 通过 `glob("_test_*.py")` + `glob("_verify_*.py")` 自动发现所有测试文件，读取首行 `# @layer:` 标记区分快慢层：

```bash
python tests/run_all_tests.py              # 默认：只跑 unit + contract（~10s）
python tests/run_all_tests.py --all         # 全量：含 integration/e2e（~25s）
python tests/run_all_tests.py --layers unit # 按层过滤
```

**新增测试零配置**：文件放入 `tests/`，命名 `_test_xxx.py`，第一行写 `@layer`——下次 `run_all_tests.py` 自动发现，无需修改任何配置。

## 三、research/ — 探索与分析

### 入目录标准

1. 目的是 **分析、探索、实验、优化**，而非验证
2. 运行结果需要**人解读**（图表、扫描结果、对比报告）
3. 没有固定的"通过/失败"语义

### 包括

- 因子研究脚本（IC 分析、因子合成实验）
- 参数扫描脚本（网格搜索最优阈值）
- 回测对比分析脚本（多策略横向对比）
- 一次性诊断脚本（数据质量检查、异常排查）
- **可复用的分析库**（`factor_lab.py`、`backtest_analyzer.py` 等）
- 从 `tests/` 清退出来的非测试脚本

## 四、results/ — 回测产出数据

### 入目录标准

1. **不包含任何可执行代码**（.py）
2. 是回测/扫描/验证管道的 **产出**，不是输入
3. 应可**随时删除重建**（通过重新运行回测）

### 结构约定

```
results/
├── index.json                     ← 所有实验的索引
├── {实验名}/
│   ├── index.json                 ← 该实验下所有 run 的索引
│   └── run_{时间戳}/
│       ├── report.json            ← 绩效指标（Sharpe, 最大回撤等）
│       ├── net_values.csv         ← 每日净值序列
│       ├── trades.json            ← 交易记录
│       ├── git_info.json          ← 运行时的 Git commit
│       └── 回测报告.md             ← 人类可读报告（可选）
```

### 不入 results/

- 任何 `.py` 脚本
- 分析中间产物（临时 CSV）
- 非回测产出的截图/图表（应放其他位置）

## 五、归属决策速查

```
文件是新的是 Py 脚本？
  ├─ 没有 assert → 不是测试 → research/
  ├─ 有 assert 但不能独立跑（是被 import 的）→ research/（库）
  ├─ 有 assert 且独立跑 → tests/
  │    ├─ 合成数据 + < 1s → # @layer: unit
  │    ├─ 对比两个实现 → # @layer: contract
  │    ├─ 需要网络/缓存 → # @layer: integration
  │    └─ 全周期回测 → # @layer: e2e
  └─ 是数据文件（.json/.csv）→ 一定是回测产出？→ results/

文件是新的是分析工具（扫描/对比/诊断）？
  └─ research/
```
