### 2.1 mootdx（通达信协议）

| 维度 | 详情 |
|------|------|
| **数据类型** | 日K/周K/月K/分钟K(1min~60min)、复权因子、行业/概念板块、财务数据、分时成交 |
| **实时性** | ✅ TCP直连通达信服务器，准实时 |
| **历史深度** | ~5年日K（实测 offset=1200约覆盖2021至今），分钟线长度不定 |
| **认证** | 免费，`pip install mootdx`，无需注册 |

**实测结果（2026-06-01）**：

| 测试项 | 结果 |
|--------|------|
| 安装 | ✅ `pip install mootdx` 成功 |
| 日K拉取 | ✅ `bars(symbol, frequency=9, start=0, offset=N)` 可用 |
| 日期参数格式 | ⚠️ `start` 不接受字符串日期，需用 `offset`（整数，倒数N根K线） |
| 复权精度 vs 新浪 | 🟢 **完全一致** — 510300(ETF) 219天偏差=0.0000%，600519(个股) 219天偏差=0.0000% |
| 历史深度 | ⚠️ offset=1200 仅覆盖~2021年起（约5年），远少于新浪的13年 |
| 分钟线 | 🟡 未实测（`frequency` 参数支持 0=分时/1=1min/2=5min 等，文档标注可用） |
| 财务/板块数据 | 🟡 未实测 |

> ⚠️ **重大修正**：社区广泛流传"mootdx 前复权与Wind/东方财富偏差1-3%"，但本项目实测发现 **mootdx 前复权 close 与新浪前复权 close 完全一致**（ETF+个股均如此）。社区的偏差报告可能来自特定时期/特定股票的差异，或不涉及前复权对比。**mootdx 前复权 K 线可以作为回测主力源的可靠替代。**

**本项目适用判断**：
- ✅ 分钟线回测（唯一免费可用的分钟级数据源）
- ✅ 作为K线的**备源/交叉验证源**（与新浪比对，发现异常）
- ✅ 实时行情补充
- ⚠️ 历史深度仅~5年（vs 新浪13年），不适用于2018年之前的全周期回测
- ⚠️ `start` 日期语义与 DataFeed 接口不兼容（需适配层转为 offset）
- **接入优先级**：高（P1）— 从 P2 升级为 P1，因复权精度疑虑已消除

## 三、综合数据方案设计

### 3.1 多源覆盖矩阵

按数据类型分层，每层指定**主源→备源→应急源**：

```
┌─────────────────────────────────────────────────────────┐
│                     数据类型覆盖矩阵                       │
├──────────────┬──────────────┬──────────────┬─────────────┤
│   数据类型    │   主源(P0)    │   备源(P1)    │  应急(P2)   │
├──────────────┼──────────────┼──────────────┼─────────────┤
│ 日K线(回测)  │ 新浪财经      │ mootdx       │ 腾讯财经    │
│ 日K线(实盘)  │ AutoStock    │ 新浪财经      │ mootdx      │
│ 分钟K线      │ mootdx       │ AutoStock    │ —           │
│ ETF净值      │ akshare(THS) │ —            │ —           │
│ 国债收益率    │ 中债官网      │ akshare      │ —           │
│ 股债利差     │ akshare+中债  │ —            │ —           │
│ 汇率/美债    │ akshare      │ —            │ —           │
│ QDII溢价     │ 集思录       │ —            │ —           │
│ 概念/板块热度 │ akshare(THS) │ mootdx       │ —           │
│ 资金流向     │ akshare(THS) │ —            │ —           │
│ 财务数据     │ akshare      │ —            │ tushare(P3) │
│ 研报/一致预期 │ akshare      │ tushare      │ —           │
└──────────────┴──────────────┴──────────────┴─────────────┘
```

### 3.2 降级策略

每个数据请求按 **主→备→应急** 顺序尝试，失败自动切换：

```
请求日K线数据
    │
    ├─ 尝试主源（新浪财经）
    │   ├─ 成功 → 返回数据 + 记录源标记
    │   └─ 失败（网络/空返回/超时）
    │       │
    │       ├─ 尝试备源（akshare/腾讯）
    │       │   ├─ 成功 → 返回数据 + 降级告警
    │       │   └─ 失败
    │       │       │
    │       │       └─ 尝试应急源（mootdx）
    │       │           ├─ 成功 → 返回数据 + 紧急告警
    │       │           └─ 失败 → 抛 DataSourceExhaustedError
```

**降级触发条件**：
| 条件 | 动作 |
|------|------|
| HTTP状态码非200 | 切换备源 |
| 返回空数据/空DataFrame | 切换备源 |
| 响应超时 > 10秒 | 切换备源 |
| 数据条数 < 预期的50% | 标记为可疑 + 切换备源 |
| 备源也失败 | 切换应急源 + 发送微信告警 |
| 所有源均失败 | 抛异常 + 微信告警 + 中止本次回测 |

### 3.3 数据交叉检验方案

针对视频提到的"mootdx数据不准"问题，建立系统性检验机制：

#### 3.3.1 日常检验（每次拉取数据时）

对K线数据做以下校验：

| 检验项 | 方法 | 阈值 | 动作 |
|--------|------|------|------|
| **日期连续性** | 检查交易日历差值 | 允许1-2天缺口（节假日） | 缺口>5天告警 |
| **OHLC合理性** | high≥max(open,close), low≤min(open,close) | 必须满足 | 不满足→标记异常行 |
| **跳变检测** | 相邻日 close 变化率 | >20%标记可疑 | 记录到异常日志 |
| **复权跳跃** | 前复权价格不应出现"向上跳" | 检测 negative return < -50% | 标记可能复权错误 |
| **成交量非负** | volume >= 0 | 必须满足 | 不满足→标记异常行 |

#### 3.3.2 跨源比对检验（定期/触发式）

两个独立数据源拉取**同一标的、同一日期区间**的K线，逐日比对：

```
新浪 close vs mootdx close vs 腾讯 close
    │
    ├─ 计算逐日偏差率 = |源A close - 源B close| / 源A close
    │
    ├─ 允许偏差 < 0.5%（正常市场差异）
    ├─ 偏差 0.5%-2%：标记为"轻度差异" → 日志记录
    ├─ 偏差 > 2%：标记为"显著差异" → 微信告警 + 人工复核
    │
    └─ 统计全周期一致性 = (偏差<0.5%的天数) / 总天数
        ├─ >99% → 两个源高度一致 ✅
        ├─ 95-99% → 可接受 ⚠️
        └─ <95% → 至少一个源有问题 ❌
```

**实施方式**：
- 新建 `tools/cross_validate_kline.py`：接受两个数据源、标的列表、日期区间，输出比对报告
- 纳入 `main_validate.py` 管道（已有验证框架）
- 每周自动跑一次全标的跨源比对

#### 3.3.3 基准数据快照验证

每次数据源升级或接入新源后：
1. 用固定日期区间（如2023-01-01~2023-12-31）拉取33只标的
2. 与 `tests/fixtures/` 中的基准快照对比
3. OHLC差异 < 0.1% 视为通过

from dataclasses import dataclass, field
from typing import Optional

@dataclass
class DataSourceConfig:
    """单个数据源配置"""
    name: str                          # 源名称：sina / mootdx / tencent / akshare
    feed: DataFeed                     # DataFeed 实例

    data_types: list[str] = field(default_factory=lambda: ["daily_k"])
    enabled: bool = True

class CompositeDataFeed(DataFeed):
    """
    多源组合数据源，自动降级。
    演进策略 L3.5 已规划，本次施工落地。
    """

    def __init__(self, sources: list[DataSourceConfig], notifier=None):
        self._sources = sorted(sources, key=lambda s: s.priority)
        self._notifier = notifier
        self._degradation_log: list[dict] = []

    def get_data(self, request: DataRequest) -> DataResponse:
        """按优先级尝试各数据源，失败自动降级"""
        for source in self._sources:
            if not source.enabled:
                continue
            try:
                response = source.feed.get_data(request)
                if self._validate(response):
                    if source.priority > 0:

                    return response

                continue
        raise DataSourceExhaustedError("所有数据源均不可用")
