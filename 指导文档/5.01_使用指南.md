## 一、环境准备（首次使用必做）

### 1.1 开发机

```bash
# 1. 克隆仓库
git clone https://github.com/<your-username>/quantforge.git
cd quantforge

# 2. 切换到开发分支
git checkout dev

# 3. 创建虚拟环境
python -m venv .venv

# 4. 安装依赖
.\.venv\Scripts\pip.exe install -r requirements.txt

# 5. 配置密钥（从模板复制并填值）
copy tokens\_templates\小熊同学token.py.template tokens\小熊同学token.py
copy tokens\_templates\wechat_webhook.py.template tokens\wechat_webhook.py
copy tokens\_templates\email_config.py.template tokens\email_config.py
# 然后编辑这三个文件，填入真实值

# 6. 验证
.\.venv\Scripts\python.exe -c "from tokens.小熊同学token import token; print(f'OK: {token[:3]}***')"
```

### 1.2 运行机

```bash
# 1. 克隆仓库
git clone https://github.com/<your-username>/quantforge.git
cd quantforge

# 2. 确认在 main 分支
git checkout main

# 3. 创建虚拟环境 + 安装依赖
python -m venv .venv
.\.venv\Scripts\pip.exe install -r requirements.txt

# 4. 配置密钥（同上）

# 5. 配置 Windows 计划任务
#    触发器：每周一至周五 09:25
#    操作：启动 monitor.bat
#    工作目录：项目根目录
```

## 二、操作总览

| 操作 | 入口 | 在哪台机器 | 说明 |
|------|------|-----------|------|
| 运行回测 | `main_backtest.py --preset <配置名>` | 开发机 | 用历史数据验证策略效果 |
| 核心模式回测 | `main_backtest.py --core --preset <配置名>` | 开发机 | 仅输出JSON指标，供批量研究用 |
| 更新数据缓存 | `main_preflight.py` | 开发机/运行机 | 全量检查和刷新数据缓存 |
| 启动实盘监控 | `monitor.bat` | 运行机 | 每交易日自动运行，推送交易建议 |
| 手动同步代码 | `git_pull.py` | 运行机 | 从远端仓库拉取最新代码 |
| 远程维护 | `remote_cmd.py` | 运行机 | 通过修改此文件在运行机执行一次性任务 |
| 运行验证 | `main_validate.py` | 开发机 | 运行验证管道（数据+回测+契约） |

## 三、操作详解

### 3.1 运行回测

**场景**：验证策略在历史数据上的表现。

```bash
cd quantforge
.\.venv\Scripts\python.exe main_backtest.py --preset tech_growth
```

**可用预设**（`config/strategies/roc_momentum/` 下的JSON配置）：

| 预设 | 说明 |
|------|------|
| `tech_growth` | 默认配置，科技/成长ETF池，全周期（2018至今）|
| `all_weather` | 全天候稳健配置 |
| `max_attack` | 激进进攻配置 |
| `sharp_defense` | 防御型配置 |

**核心模式**（只输出JSON指标，无图表/报告，适合批量回测）：

```bash
.\.venv\Scripts\python.exe main_backtest.py --core --preset tech_growth
```

**输出示例**：
```json
{
  "sharpe": 1.3286,
  "sortino": 2.1234,
  "total_return": 2.886,
  "max_drawdown": 0.1305,
  "calmar": 10.18,
  "trade_count": 48,
  "total_commission": 535.21
}
```

**当前默认配置（tech_growth）**：
- 策略：ROC 动量轮动
- 标的池：33只科技/成长ETF（含创业板、科创板、半导体、AI、云计算、通信、软件等）
- 回测区间：2018-01-01 ~ 至今
- 初始资金：40,000
- TOP_K：5（等权分配）
- ROC回看期：22天
- 买入阈值：20.0%
- 回撤止损：8%
- 高水位止盈：10%
- ATR扩张过滤 + ADX趋势质量：开启

**修改策略参数**：编辑 `config/strategies/roc_momentum/<preset>.json`，或创建新预设文件后通过 `--preset` 指定。

**输出位置**：
- 日志：`logs/log.txt`（INFO）、`logs/log_error.txt`（ERROR）
- 数据缓存：`data/sina/fund_data_{code}.json`、`data/sina/batch_params.json`
- 回测报告/图表：`results/` 目录

### 3.2 更新数据缓存

回测时会自动更新。如果只想检查/更新数据不跑回测：

```bash
.\.venv\Scripts\python.exe main_preflight.py
```

或手动调用：

```python
from core.data_feed import CachedDataFeed
from data_sources.sina_feed import SinaFinanceFeed

feed = CachedDataFeed(SinaFinanceFeed(), cache_dir='./data/sina')
feed.update_cache(
    codes=["159915", "510300", "510500"],
    data_type="fund",
    start="2018-01-01",
    end="2026-12-31",
)
```

**缓存策略**：
- 每个标的一个 JSON 文件
- 增量更新：只补充新数据，不覆盖已有数据
- 7天内数据已是最新则跳过

### 3.3 启动实盘监控

**自动方式**（推荐）：Windows 计划任务每交易日 09:25 自动触发 `monitor.bat`。

**手动方式**：

```bash
cd quantforge
monitor.bat
```

**monitor.bat 执行链路**：

```
monitor.bat
  ├── Step 1: git_pull.py        ← 从远端仓库拉取最新代码
  ├── Step 2: remote_cmd.bat     ← 执行远程维护命令
  ├── Step 3: main_monitor.py    ← 核心监控主程序
  │     ├── 判断是否交易日
  │     ├── 非交易日 → 微信通知 → 退出
  │     └── 交易日 → 多进程并行启动各策略监控
  └── Step 4: send_error_log.py  ← 收集并发送错误日志
```

**main_monitor.py 当前状态**：框架已就绪，`monitor_tasks` 列表为空。添加策略监控任务后即可运行：

```python
# main_monitor.py 中的 monitor_tasks
monitor_tasks = [
    (roc_strategy_monitor, (wechat, email), "ROC动量轮动"),
    # 添加更多策略...
]
```

### 3.4 手动同步代码

运行机从远端仓库拉取 main 分支最新代码：

```bash
.\.venv\Scripts\python.exe git_pull.py
```

- 最多重试 10 次，每次失败等待 30 秒
- 成功后微信通知更新信息（含新增提交数量和提交详情）
- 无新提交也会通知

### 3.5 远程维护

**场景**：需要在运行机上执行一次性任务（移动日志、更新持仓文件等），但无法直接操作运行机。

**操作步骤**：

1. 在开发机上编辑 `remote_cmd.py`，添加维护任务
2. 提交并推送到 main 分支
3. 运行机下次启动时自动执行

```python
# remote_cmd.py 示例
def remote_cmd():
    add_content('开始执行 remote_cmd.py')

    # 示例：在指定日期执行策略持仓更新
    # target_date = date(2026, 3, 30)
    # if date.today() == target_date:
    #     edit_strategy_file(...)

    add_content('remote_cmd.py 执行完毕')
```

执行结果通过微信通知发送。

### 3.6 修复计划任务服务

Windows 计划任务服务偶尔会异常停止，导致监控不触发：

```bash
fix_task.bat
```

检查 `Schedule` 服务是否运行，未运行则尝试启动。

## 七、核心概念

### 7.1 数据流

```
DataFeed（数据层）
  → Strategy.produce_decisions()（策略层，产出Decision）
    → Resolver.resolve()（组合层，Decision→TargetPosition）
      → Executor.execute()（执行层，模拟交易或推送通知）
```

### 7.2 Decision ≠ 交易指令

策略产出的 Decision 是语义丰富的决策对象，不是简单的买/卖信号：

| DecisionType | 含义 | Resolver |
|---|---|---|
| ROTATION | 多标的轮动，产出排序+权重 | RankingResolver（TOP_K+权重分配） |
| TIMING | 单标的择时，产出入场/出场 | TimingResolver（enter=全仓，exit=清仓） |

### 7.3 回测 vs 实盘

同一套策略代码，区别只有两处：

| | 回测 | 实盘 |
|---|---|---|
| DataFeed | CachedDataFeed(SinaFinanceFeed)（新浪缓存） | AutoStockFeed（实时API） |
| Executor | BacktestExecutor（模拟撮合） | LiveExecutor（推送通知，不自动下单） |

## 八、常见问题

**Q: 运行回测报 `API_TOKEN 未设置`？**
A: 回测使用新浪财经数据源，不需要 token。只有实盘监控（AutoStockFeed）才需要 `tokens/小熊同学token.py`。

**Q: 运行机监控没有触发？**
A: 1) 检查 Windows 计划任务是否配置正确；2) 运行 `fix_task.bat` 检查服务状态；3) 查看 `logs/runtime_error.txt`。

**Q: 如何添加新策略？**
A: 1) 在 `strategies/_configs/` 创建配置类；2) 在 `strategy/factory.py` 注册；3) 在 `config/strategies/` 创建 JSON 预设；4) 在 `strategies/` 创建策略类。

**Q: 如何修改ROC策略参数？**
A: 编辑 `config/strategies/roc_momentum/<preset>.json`，或复制创建新预设后通过 `--preset` 指定。不要直接修改 `strategies/_configs/roc_config.py` 中的默认值（那是代码级后备，应通过JSON覆盖）。

**Q: 如何快速验证策略改动？**
A: 使用 `--core` 模式仅输出核心指标JSON，比完整报告快很多：`main_backtest.py --core --preset tech_growth`

**Q: 回测结果可靠吗？能和实盘一致吗？**
A: 策略决策和信号解析逻辑完全一致（经契约测试验证）。区别仅在于：1) 回测使用收盘价，实盘14:40执行（约等于收盘价）；2) 实际滑点约0.1%。成本敏感性分析证实滑点对Sharpe影响约-0.033（2.5%），回测Sharpe≈1.33基本等于实盘可预期水平。详见 [5.13_全周期参数稳健性重验证.md](./5.13_全周期参数稳健性重验证.md)。

**Q: 数据缓存太大？**
A: `data/sina/` 目录可安全删除，下次运行回测时会自动重新拉取。
